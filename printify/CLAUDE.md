# printify/ — Printify -> Etsy publisher (printify-publisher's domain)

Owns the bridge from finished gallery art to live Etsy product listings.

## Mission

When an image lands in `site/src/assets/gallery/<category>/`, generate two
Printify products (Poster + Wrapped Canvas) and publish them into the user's
existing Etsy shop via Printify's Etsy sales channel. Keep an idempotent ledger
so the same image is never published twice.

## Scope

**In:**
- Printify REST v1 API client (`printify_api.py`)
- Bootstrapping config from the *existing* Etsy shop products
- Idempotent publish orchestrator with dry-run-first discipline
- Ledger of published products (committed to git as audit trail)

**Out:**
- Generating Midjourney prompts (that's `mj/`)
- Picking ref images (that's `refs/`)
- Modifying the Astro site or `products.json` (that's `site-custodian` /
  `tools/sync_gallery.py`)
- Writing to the DB (`db/dino_art.db`)
- Driving the launchd watcher itself (lives in `tools/`)

## Sales channel decision (locked)

**Path A — Printify -> Etsy.** Printify manages product listings inside the
user's existing Etsy shop. We do NOT use Printify Pop-Up. All publishes target
the Etsy-channel shop returned by `GET /v1/shops.json`.

## "Existing Etsy products are the source of truth" protocol

User's quote: *"if you ever need any reference for printify use the products
that are already in the store, they are the right size and price."*

Every config field that talks about blueprints, providers, variant ids, or
prices must be **derived from a live shop scan**, not from documentation.

`--bootstrap-config` does the discovery:

1. `GET /shops.json` -> pick the Etsy shop (or `PRINTIFY_SHOP_ID` env override).
2. `GET /shops/{id}/products.json` (paginated) -> collect every existing product.
3. For each product, `GET /shops/{id}/products/{pid}.json` -> read variants
   with their live retail prices.
4. Classify by title: "canvas" -> wrapped_canvas, else poster.
5. For each (kind, size) bucket, capture the first match's blueprint_id,
   print_provider_id, variant_id, and retail_cents.
6. Write the result to `printify_config.yaml`.

Re-run any time the user adds/edits products in Printify or Etsy.

## Dry-run / live discipline

- Default flag is `--dry-run`. **Real publish requires explicit `--live`.**
- A dry-run prints the exact API endpoints + JSON payloads that *would* fire,
  reads costs/products from Printify, and writes nothing.
- Live publish creates one Printify product per kind, then calls
  `POST /shops/{id}/products/{pid}/publish.json` to push to Etsy.
- After a successful `--live` for an image, the ledger is updated **before**
  moving to the next image, so an interrupted run resumes cleanly.

## Ledger format

`printify/printify_ledger.json`:

```json
{
  "version": 1,
  "entries": {
    "predators/tyrannosaurus-rex-apex-predator-roar.png": {
      "published_at": "2026-04-29T22:14:08+00:00",
      "image_id": "<printify upload id>",
      "products": [
        {
          "kind": "poster",
          "product_id": "<printify product id>",
          "external_url": "<etsy listing handle>",
          "variants": [
            { "id": 33742, "size": "12x18", "price": 2499 },
            { "id": 33743, "size": "18x24", "price": 3499 },
            { "id": 33744, "size": "24x36", "price": 4999 }
          ]
        },
        {
          "kind": "wrapped_canvas",
          "product_id": "...",
          "external_url": "...",
          "variants": [ ... ]
        }
      ]
    }
  }
}
```

Idempotency rule: if `entries[<gallery-relative-path>]` exists, skip.

## Cross-domain contracts

**Reads:**
- `site/src/data/products.json` for SEO title / description / keywords / category
- `site/src/assets/gallery/<category>/*.png` for the source artwork
- `.env` at repo root for `PRINTIFY_API_KEY` (and optional `PRINTIFY_SHOP_ID`)

**Writes (only):**
- `printify/printify_ledger.json`
- `printify/printify_config.yaml` (only with `--bootstrap-config`)

Never writes to `site/`, `tools/`, `db/`, `refs/`, or `mj/`.

## Pricing rules

1. **Match existing product prices** when the same (kind, size) variant is
   present in the shop. This is the primary path.
2. **Fallback:** `retail = provider_cost * 2.2`, then round up to nearest
   `X.99`. Used only when bootstrap finds no existing price for that size.
3. **Free shipping override** on every published product.

## Image fit (Pillow)

When the source MJ image's aspect ratio differs from the target print size:
- If cropping loss <= 15%, center-crop.
- Else pad with `edge_median` background sampling, then crop.
- Decision is logged in the ledger entry for that image.

(Implementation lands when first --live publish is attempted; dry-run does not
need to crop.)
